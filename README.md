#ysyx_repository 

## 🛠️ 工程开发速查表 (Git & 终端)

### 1. 日常开发三步曲 (最高频操作)
这是每次完成一个功能模块（如写完一个 Verilog 文件或 C 程序）后必做的操作。

* **第一步：看雷达**
  `git status` (查看哪些文件被修改了)
* **第二步：送入暂存区**
  `git add .` (把当前目录下所有非黑名单的修改全部暂存)
  `git add adder.v` (如果只想暂存特定的文件，写上文件名)
* **第三步：永久存档并推上云端**
  `git commit -m "feat: 这里写清楚你做了什么修改"` (本地生成快照)
  `git push` (发射到 GitHub)

---

### 2. 时光机：版本回退与撤销
硬件调试难免搞炸系统，学会使用“后悔药”。

* **清空当前未存档的草稿 (极其常用)**
  `git checkout .` (将工作区所有修改恢复到上次 commit 的状态，**未保存的新代码会丢失！**)
* **查看历史坐标**
  `git log --oneline` (查看历史快照的 ID 列表)
* **回到过去看看 (观光模式)**
  `git checkout <快照ID>` (比如 `git checkout a1b2c3d`)
  `git checkout main` (看完后，切回当前的最新进度)
* **彻底推倒重来 (危险操作)**
  `git reset --hard <快照ID>` (将整个系统强制回退到指定节点，**该节点之后的记录将灰飞烟灭**)

---

### 3. Git 环境与诊断
* `git ls-files` (查看当前被 Git 追踪的所有文件，检查有没有垃圾文件混进来)
* `git config --list` (查看当前配置的 GitHub 用户名和邮箱)

---

### 4. 终端导航指令 (Git Bash 必会)
* `pwd` (打印当前所在目录的绝对路径，防迷路)
* `ls -a` (查看当前目录下的所有文件，包括隐藏的 `.git` 和 `.gitignore`)
* `cd ..` (返回上一级父目录，注意中间有空格)
* `cd ~` (一键传送回系统的用户根目录)
